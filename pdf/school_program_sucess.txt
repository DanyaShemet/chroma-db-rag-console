Навчальна програма з програмування (12–16 років)
Загальна мета

Формування алгоритмічного мислення, базових навичок програмування та розуміння принципів створення цифрових продуктів із поступовим ускладненням матеріалу та розвитком самостійності учнів.

Рік 1: Основи програмування та алгоритмічного мислення
Учні повинні опанувати:
базові поняття:
змінні, типи даних
умовні оператори
цикли
базові алгоритмічні структури
основи роботи з рядками та масивами
поняття функцій
Учні повинні вміти:
писати прості програми (Python / JavaScript)
розв’язувати задачі на логіку
читати та пояснювати код
Додатково:
базове знайомство з:
налагодженням (debugging)
помилками (syntax vs logic)філ
Проєкти:
калькулятор
гра “вгадай число”
простий текстовий чат

Рік 2: Структури даних та прикладне програмування
Учні повинні опанувати:
списки, словники
базові алгоритми:
пошук
сортування (простий рівень)
введення/виведення даних
основи роботи з файлами
вступ до ООП (класи, об’єкти)
Учні повинні вміти:
структурувати код
працювати з даними
розбивати задачу на підзадачі
Додатково:
вступ до веб:
HTML (основи)
CSS (базовий стиль)
базове розуміння як працює клієнт-сервер
Проєкти:
To-do list
простий веб-сайт
гра з графікою (наприклад, на canvas)
Рік 3: Розробка застосунків та сучасні технології
Учні повинні опанувати:
JavaScript глибше або Python
бази даних (SQLite або простий MongoDB)
API (що це таке і як працює)
основи Git
Учні повинні вміти:
створювати прості повноцінні застосунки
працювати з API
використовувати бібліотеки
Додатково:
вступ до:
асинхронності
базової безпеки (не зберігати паролі відкрито)
знайомство з AI-інструментами:
як допоміжний інструмент, а не заміна мислення
Проєкти:
веб-додаток (frontend + простий backend)
чат або форма збереження даних
невеликий командний проєкт
Підсумковий проєкт

Учні створюють застосунок, який:

має інтерфейс (UI)
працює з даними
вирішує реальну задачу

(наприклад: планувальник, міні-гра, навчальний сервіс)

Система оцінювання

Оцінювання базується на:

розумінні матеріалу
якості коду (читабельність, структура)
здатності пояснити рішення
участі в проєктах

Враховується:

процес навчання, а не лише результат
Підхід до навчання
поєднання теорії та практики
поступове ускладнення матеріалу (scaffolding)
робота в парах і командах
підтримка та зворотний зв’язок від викладача
